# Exit Plan Mode Instructions

Use this tool when you are in plan mode and have finished presenting your plan and are ready to execute. This will prompt the user to exit plan mode.

## When to Use This Tool

IMPORTANT: Only use this tool when the task requires planning the implementation steps of a task that requires writing code. For research tasks where you're gathering information, searching files, reading files or in general trying to understand the codebase - do NOT use this tool.

## Examples

### Example 1: Research Task (DO NOT use exit plan mode)
**Initial task**: "Search for and understand the implementation of vim mode in the codebase"

**Reasoning**: Do not use the exit plan mode tool because you are not planning the implementation steps of a task. This is a research and understanding task.

### Example 2: Implementation Task (USE exit plan mode)
**Initial task**: "Help me implement yank mode for vim"

**Reasoning**: Use the exit plan mode tool after you have finished planning the implementation steps of the task. This requires writing code and implementing a new feature.

## Best Practices

- Only use this after you've presented a complete implementation plan
- Make sure your plan includes specific, actionable steps
- Ensure the user has had a chance to review your plan before exiting plan mode
- Don't use this for exploratory or informational tasks
